How to use this page
  • Start with Easy, then Medium, then Hard — each tier assumes the patterns from the tier before it
  • Every page follows the same 6 steps: Requirements → Core Entities → API → Data Flow → High-Level Design → Deep Dives
  • Read the flash card at the top of each page first — it's the 5 things a strong candidate says in the first 2 minutes
  • Practice delivery out loud with a timer — the time badge on each card is the realistic interview length
  • Come back to the Follow-up Q&A section after a first pass once the core design is solid
